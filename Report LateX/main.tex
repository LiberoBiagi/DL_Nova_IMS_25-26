\section{Introduction}

The classification of artistic images poses unique challenges compared to conventional image recognition tasks. 
Paintings often exhibit significant variability in style, composition, and abstraction, even within the same artist class, while distinct classes may share similar visual characteristics.
 As a result, effective models must capture subtle patterns such as texture, color distribution, and stylistic features that are commonly learned through deep convolutional representations \cite{cetinic2018finetuning,li2024artimage}.

This project addresses the problem of classifying artworks from a subset of the WikiArt dataset using deep learning techniques. 
The objective is to develop models capable of learning discriminative visual representations that generalize well despite the complexity and variability of the data.

To this end, several deep learning approaches are explored and evaluated, with a focus on understanding how different architectures capture the underlying structure of artistic images. 
Convolutional neural networks have been shown to be effective at learning hierarchical feature representations from images \cite{cetinic2018finetuning}. 
The performance of these models is assessed using standard evaluation metrics, providing insight into their effectiveness for this task.


\section{Dataset and Data Preparation}
\subsection{Dataset Description}

The dataset used in this project is a subset of the WikiArt collection, containing 13,340 paintings from 23 artists. 
Each image is labeled by its creator, forming a multi-class classification task. 
The dataset is moderately imbalanced, with the number of images per artist ranging from 336 to 1322, which introduces additional challenges for model training and evaluation.
All images have a uniform resolution of 512×512 pixels. 
Exploratory analysis further shows that artists differ in color composition, including variations in grayscale usage and distinct RGB/HSV distributions, indicating that color-related features are likely informative for classification.


\subsection{Data Splitting Strategy}

To ensure that the model generalizes across different visual styles of each artist and to address the class imbalance, we adopted a clustering-based splitting strategy within each artist. 
For each image, we extracted low-level visual features, including a grayscale indicator and statistical descriptors in HSV color space, after resizing images to $64 \times 64$ and normalizing pixel values.
Clustering was performed independently per artist using KMeans, with the number of clusters selected by maximizing the silhouette score. 
This groups artworks with similar visual characteristics (e.g., color palette or style) within each artist.

Initially, we experimented with a purely random split by artist, but observed imbalanced results. 
This led to situations where the model was not exposed during training to certain stylistic variations, negatively affecting performance. 


The final dataset split was obtained by stratifying on the combined artist–cluster label, using an 80/10/10 ratio for training, validation, and testing. 
This ensures that each split contains samples from all clusters of every artist, exposing the model to the full range of styles during training.
The ratio was chosen to maximize the number of training samples while maintaining sufficiently large and balanced validation and test sets.

Stratification by artist–cluster helps preserve both class distribution and intra-class variability across splits, leading to a more robust and fair evaluation of the classification model.

\subsection{Data Preprocessing}

Prior to training, images were preprocessed to ensure compatibility with different model architectures. 
For the baseline model, pixel values were normalized from the range [0,255] to [0,1] and images were resized to 224x224 pixels, while pretrained models used their respective input preprocessing requirements. 

Several data augmentation techniques were evaluated, including random rotation, horizontal flipping, zoom, and contrast adjustments, both individually and in combination. 
However, experimental results showed that augmentation did not improve performance. 
Consequently, data augmentation was not included in the final training pipeline.

The dataset was processed in batches of 64 samples, providing a balance between memory efficiency and training performance.

\section{Methodology}

\subsection{Custom CNN Architecture}

The proposed CNN follows a design in which the number of filters increases progressively across convolutional blocks (32, 64, 128, 256). 
This allows the network to learn increasingly complex representations: early layers capture low-level features such as edges and colors, while deeper layers encode higher-level patterns such as textures and stylistic characteristics.
The increase in filter depth also compensates for the reduction in spatial resolution caused by pooling operations.

All convolutional layers use $3 \times 3$ kernels with \texttt{same} padding, preserving spatial dimensions and allowing the network to keep extracting features at the same resolution before downsampling is applied later via pooling.

Instead of using a flattening operation, we apply Global Average Pooling at the end of the convolutional stack. 
This reduces each feature map to a single value, drastically lowering the number of parameters.

The final classifier consists of two fully connected layers with dropout regularization, followed by a softmax output layer for multi-class prediction.

This architecture was achieved by trial and error until performance plateaued at an F1-score of 0.58. 
Furthermore, we applied grey scaling and obtained significantly worse results, leading to the conclusion that our model used mainly color as a discriminative feature.
As a result, we decided to adopt transfer learning with pre-trained models in order to capture more complex features such as shapes, textures, and corners. 


\subsection{Transfer Learning with Pre-trained Models}
For all architectures described below, pretrained weights from ImageNet were used and the backbone networks were treated as fixed feature extractors, with their layers frozen during training. 
A custom classification head was added on top to adapt each model to the artist classification task.

\subsubsection{ResNet50}

ResNet50 is a deep convolutional neural network architecture composed of 50 layers, based on the concept of residual learning. 
Its key feature is the use of residual connections, which allow the network to learn residual mappings instead of direct transformations. 
These connections facilitate the learning of both low-level and high-level features, improving the model’s capacity to generalize across varying artistic styles and making ResNet50 a suitable choice for our problem.


\subsubsection{InceptionV3}

InceptionV3 is a convolutional neural network architecture designed to capture visual features at multiple scales. 
Its main idea is the use of Inception modules, where multiple operations (e.g., $1 \times 1$, $3 \times 3$, $5 \times 5$ convolutions and pooling) are applied in parallel to the same input and their outputs are concatenated. 
This allows the model to extract different types of features simultaneously.
This architecture is particularly useful when the size of relevant patterns is unknown, as it can capture both fine details and more global structures. 
For our task, this is beneficial since artistic styles can vary in both local features and overall composition.


\subsubsection{Vision Transformer (ViT)}

ViT is an architecture that applies transformer models to image classification. 
Instead of using convolutions, the image is divided into fixed-size patches, which are treated as a sequence and processed using self-attention mechanisms, allowing the model to capture global relationships between different parts of the image.
This is useful for artist classification, where overall composition and spatial relationships can be important.


\section{Experimental Setup}

All models were trained using the Adam optimizer with a learning rate of $10^{-3}$ and the sparse categorical cross-entropy loss function, which is suitable for multi-class classification with integer-encoded labels.

Model performance was primarily evaluated using the weighted F1-score, as it provides a better assessment under class imbalance, although accuracy was also monitored during training.

Training was performed for a maximum of 60 epochs. 
To prevent overfitting and reduce unnecessary computation, early stopping was applied based on the validation loss, with a patience of 10 epochs for CNN-based models and 5 epochs for the Vision Transformer. 
The best model weights were restored after training.

Additionally, a learning rate scheduler (ReduceLROnPlateau) was used to reduce the learning rate by a factor of 0.5 when validation loss plateaued, with a minimum learning rate of $10^{-6}$.
Model checkpoints were saved based on the best validation loss.

\section{Results and Evaluation}
\subsection{Model Comparison}

Appendix~\ref{tab:model_comparison} presents the performance of all evaluated models.

ResNet50 achieved the best overall performance across all metrics, with the highest F1-score and validation accuracy, while maintaining a reasonable computational cost. 

InceptionV3 showed competitive but lower performance, whereas the custom CNN performed significantly worse, highlighting the benefits of transfer learning in capturing complex artistic features. 

The ViT performed substantially worse than all other models, with lower accuracy and higher loss, and also required significantly longer training time.

For these reasons, ResNet50 was selected as the baseline model for further analysis.

\subsection{Training Behavior of ResNet50}

Appendix~\ref{ResNet50_plots} shows the training and validation loss and accuracy curves for ResNet50.
The model exhibits a steady decrease in training loss and a corresponding increase in training accuracy, indicating effective learning. 
Validation performance improves rapidly during the initial epochs and stabilizes after.
A noticeable gap between training and validation curves emerges as training progresses, with training accuracy reaching approximately 97\% while validation accuracy plateaus around 79\%. 
Similarly, training loss continues to decrease, whereas validation loss stabilizes around 0.79, which indicates moderate overfitting.
Despite this, the validation performance remains strong, suggesting that the model generalizes well overall. 
The use of early stopping and learning rate scheduling helps prevent further overfitting and ensures stable convergence.

\subsection{Per-Class Performance of ResNet50}

A per-class analysis of ResNet50 reveals variability in performance across artists (see Appendix~\ref{tab:class_report}). 
Several classes achieve high F1-scores, indicating that the model effectively captures distinctive stylistic features. 
In contrast, lower performance is observed for certain classes (e.g., classes 15 and 21), where F1-scores drop below 0.6.
This variation can be attributed to two main factors. 
First, class imbalance affects performance, as classes with fewer samples tend to be harder to learn. 
Second, some artists exhibit similar visual styles, making them more difficult to distinguish. 
Overall, while the model performs well on average, these results highlight the challenges of intra-class variability and inter-class similarity in artist classification.

\section{Fine-tuning of ResNet50}
Two rounds of adjustments to the training procedure of ResNet50 were performed.

In the first round, the learning rate was decreased to $10^{-5}$, the number of epochs was reduced by 10, the minimum learning rate was set to $10^{-7}$, and the last 30 non-batch normalization layers were unfrozen.
This approach is more conservative, as it only adjusts the highest-level features of the model while keeping most of the pre-trained knowledge unchanged.

In the second round, the learning rate was decreased to $10^{-6}$, the number of epochs was reduced by 10, the minimum learning rate was set to $10^{-8}$, and the last 60 non-batch normalization layers were unfrozen.
This approach is more aggressive, as it unfreezes a larger number of layers, allowing the model to adjust a larger portion of its parameters, potentially capturing more task-specific features but also increasing the risk of overfitting and instability.

Both strategies reached a plateau in performance at epoch 12, with an F1-score of 0.79, which is the same as the baseline model with frozen layers.


\subsection{Deeper ResNet}

To improve the model's representational capacity, we introduced a sequence of three dense layers instead of a single classification layer.
The ResNet backbone was frozen and only the added layers were trained, with the same experimental setup as before.
The deeper classifier achieved an F1-score of 0.76, which is slightly lower than the baseline model.
Two rounds of fine-tuning were applied (as described above), reaching a plateau at an F1-score of 0.78 at epochs 11 and 14, respectively.

\section {Final Model and Interpretation (Grad-CAM)}

Based on the results of the previous experiments, we selected the deeper ResNet with the second round of fine-tuning as our final model, as it has a stronger generalization
capacity while maintaining comparable performance to the baseline.

The confusion matrix (see Appendix~\ref{confusion_matrix}) shows a strong diagonal, indicating that the model correctly classifies most samples across classes. 
Misclassifications are concentrated among specific groups, such as classes 8 and 21, which exhibit lower true positives and are confused with multiple other classes.
Moreover, class 8 was predicted as class 1 a considerable number of times, indicating that these two artists may share similar visual characteristics that the model struggles to differentiate.
In contrast, classes like 14, 17, and 22 achieve very high correct predictions, indicating that these categories have more consistent and recognizable patterns. 

The Grad-CAM analysis provides insight into how the model makes its predictions across different artists.
The best predicted class was Ivan Aivazovsky, which can be attributed to the strong consistency of his work. 
His paintings are predominantly seascapes with similar perspectives and balanced color compositions, making them easier for the model to recognize. 
The attention maps confirm this, as the model focuses on key elements such as the horizon between sky and sea, clouds, and ship features (see Appendix~\ref{Ivan_Aivazovsky}).
 
In contrast, the worst-performing class was Salvador Dalí, whose work spans a wide range of styles and color schemes. 
Grad-CAM visualizations show that the model’s attention is scattered across the image, indicating uncertainty and difficulty in identifying consistent features (see Appendix~\ref{Salvador_Dalí}). 



\section{Discussion}

The strong performance of ResNet50 is likely due to the effectiveness of residual connections and pretrained convolutional features, which allow the model to learn rich visual representations even with a relatively limited dataset. 


InceptionV3 also benefits from transfer learning and multi-scale feature extraction, but its more complex architecture may make optimization slightly less stable in this setting.

The custom CNN underperforms compared to pretrained models, which is expected given that it must learn all feature representations from scratch without large-scale pretraining. 

The ViT performs worst, which can be attributed to its reliance on large-scale data and lack of built-in inductive biases, i.e., it does not inherently exploit local spatial patterns as convolutional models do. 
With a smaller dataset and without fine-tuning, it struggles to learn effective representations.

Fine-tuning experiments did not lead to further improvements, as performance consistently plateaued around the baseline, likely because the ResNet50 backbone already captures the most
relevant features, and further improvements may require more extensive adjustments or additional data to be effective.
However,  even though our baseline had better results, it was mildly overfitting. 
For this reason, we decided to implement the deeper ResNet with the second round of fine-tuning as our final model.
It can also be argued that deeper models are better suited for problems like ours, rather than shallow ones \cite{cetinic2018finetuning}.

The confusion matrix indicates strong overall performance, with misclassifications occurring mainly between a limited number of classes.

The Grad-CAM analysis indicates that the model performs better on artists with stable visual patterns and struggles when stylistic variability is high.

\section{Conclusion}

The results of this project show that transfer learning with pretrained architectures, particularly ResNet50, significantly outperforms models trained from scratch, highlighting the importance of learned visual representations.
The model performs well on artists with consistent styles but struggles with those exhibiting high variability, as confirmed by confusion matrix analysis and Grad-CAM visualizations.
Overall, while strong performance was achieved, this work highlights the challenges of artistic image classification and points to future directions such as larger datasets and improved feature modeling.
